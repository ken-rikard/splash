    IMPORTANT — Pitfall 1 merge guard: In the REST fetch handler, use functional update:
    ```
    setAlerts(prev => {
      const serverIds = new Set(data.map(a => a.riverId))
      const extras = prev.filter(a => !serverIds.has(a.riverId))
      return [...data.map(a => ({ ...a, triggeredAt: new Date(a.triggeredAt) })), ...extras]
    })
    ```
    This prevents the race condition where an SSE alert-trigger arrives before the REST response completes.
