<objective>
Implement FlowStore, typed EventBus, RiverRegistry (persistence), and NveHydApiAdapter.

Purpose: Build the four core runtime components. FlowStore holds in-memory flow data, EventBus publishes typed events, RiverRegistry provides persistent CRUD for river metadata, and NveHydApiAdapter fetches discharge data from NVE HydAPI.

Per D-15 (FlowStore) — in-memory Map<id, RiverData>, no disk persistence for flow.
Per D-16 (EventBus) — typed EventEmitter with data-update, error, stale, status-change events.
Per D-13/D-14 (RiverRegistry) — persists RiverEntry to data/rivers.json.
Per D-02/D-03 — NveHydApiAdapter uses native fetch targeting NVE HydAPI Observations endpoint.
Per D-03b — Adapter skips gracefully if NVE_API_KEY not set.
Per D-11 — Adapter returns typed RiverData[]; raw JSON never leaks past parser boundary.
</objective>
