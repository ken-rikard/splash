<objective>
Implement the in-memory data store, typed event bus, and the primary datasource adapter for hvorerdetvann.com.

Purpose: Build the three core runtime components that the scraper engine orchestrates. The DataStore holds scraped results, the EventBus publishes typed events for Phase 2's UI to consume, and the HvorErDetVannAdapter demonstrates the full DatasourceAdapter contract.

Per D-13 — DataStore is in-memory Map<id, RiverData> with no disk persistence.
Per D-14 — ScraperEventBus wraps Node.js EventEmitter with typed event map.
Per D-02/D-03 — HvorErDetVannAdapter uses native fetch (not node-fetch) targeting /api/sections JSON API.
Per D-11 — Adapter returns typed RiverData[]; raw JSON never leaks past parser boundary.
</objective>

      - `ApiResponse`: `sections: ApiSection[]`
    - Define `ZONE_TO_LEVEL: Record<string, AlertLevel>`: dry→1, low→2, medium→3, high→4, very_high→5
    - Define `parseApiSection(section: ApiSection): RiverData`:
      - id = `hvorerdetvann:${section.section.id}`
      - name = section.section.name
      - source = 'hvorerdetvann'
      - currentLevel = section.last_flow?.flow ?? null
      - alertLevel = ZONE_TO_LEVEL[section.zone] ?? 0 (fail-safe: 0 signals unknown)
      - lastUpdated = new Date(section.last_flow.timestamp)
      - status = 'ok'
    - Export class `HvorErDetVannAdapter implements DatasourceAdapter`:
      - `readonly sourceId = 'hvorerdetvann'`
      - Private `baseUrl = 'https://hvorerdetvann.com/api/sections'`
      - `async fetch(): Promise<RiverData[]>`:
        - response = await fetch(this.baseUrl, { headers: { Accept: 'application/json' } })
        - if !response.ok, throw Error with status text
        - const data = await response.json()
        - if !data?.sections || !Array.isArray(data.sections), throw new TypeError with message containing 'structure' and 'missing sections array' (so engine can identify fatal errors per D-07)
        - return data.sections.map(parseApiSection)
    - The adapter does NOT do retry — retry is the engine's responsibility (Plan 03 per D-05).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External API → HvorErDetVannAdapter | Untrusted JSON response crosses into the parser boundary. Must validate structure before creating RiverData objects. |
| Adapter → DataStore | Typed RiverData[] passes internal boundary. No raw JSON crosses this boundary per D-11. |
| EventBus → Consumers | Typed events carry only RiverData[] and status objects. No untrusted data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | HvorErDetVannAdapter.fetch() | mitigate | Runtime type guard checks `data?.sections` is an array with optional chaining. Missing/changed structure throws TypeError per fatal-error semantics D-07. |
| T-02-02 | Denial of Service | HvorErDetVannAdapter.fetch() | accept | Adapter does not retry — engine handles retry in Plan 03. Single fetch call has no resource-exhaustion vector beyond normal network I/O. |
| T-02-03 | Denial of Service | EventEmitter listener leak | mitigate | ScraperEventBus provides `removeListener` method. Tests verify removeListener stops event delivery. |
| T-02-04 | Information Disclosure | Test fixtures | accept | All fixture data is synthetic or derived from public API sample. No secrets or credentials. |
| T-02-SC | Tampering | npm dependencies | mitigate | p-retry, node-crypt, vitest, tsx all Verified in RESEARCH.md Package Legitimacy Audit. No postinstall scripts. |
</threat_model>
