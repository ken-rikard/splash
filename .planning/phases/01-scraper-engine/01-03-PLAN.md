<objective>
Build the engine orchestrator and application entry point with cron-based scheduling and river registry integration.

Purpose: Tie together FlowStore, EventBus, RiverRegistry, and NveHydApiAdapter from Plan 02 into a cohesive ScraperEngine. The engine handles retry logic (p-retry), stale detection, error state management, adapter lifecycle, and loads enabled station IDs from the river registry.

Per D-05 — Exponential backoff retry: base 2s, cap 60s, max 3 retries via p-retry.
Per D-06 — Stale-data threshold: 30 minutes since last successful fetch.
Per D-07 — Non-fatal errors retry silently; fatal errors (TypeError) abort retry via pRetry.AbortError.
Per D-08 — Error state is stored in RiverData.status for Phase 2 UI.
Per D-10 — Adapters registered via engine.register(); engine iterates and consolidates.
Per D-04 — node-cron schedule with env-var-driven cron expression, default every 15 minutes.
Per D-13/D-14 — Engine loads RiverRegistry on startup to get enabled station IDs.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External API → ScraperEngine (via NVE adapter) | Untrusted external data enters the engine. Validation happens in adapter parse(). |
| Engine → FlowStore | Only typed RiverData[] stored. Engine sets status to 'ok', 'stale', 'error'. |
| Engine → RiverRegistry | Reads enabled station IDs. Only trusted file I/O. |
| Engine → EventBus | Typed events only. No raw external data crosses. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | DoS | fetchWithRetry() | mitigate | Max 3 retries, exp backoff cap 60s. allSettled prevents blocking. |
| T-03-02 | DoS | retry loop on failure | mitigate | TypeError (fatal) → AbortError skips retry. After 3 failed, error state. |
| T-03-03 | Tampering | Stale data served silently | mitigate | D-06 staleness window enforced. 'stale' event emitted. Pitfall: null lastUpdated never stale. |
| T-03-04 | EoP | Event listener injection | accept | Typed listeners only. Same process only. |
| T-03-05 | Repudiation | Scrape failures not detectable | mitigate | Every scrape emits status-change. Error captured in error field. |
</threat_model>
