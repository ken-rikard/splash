## Web Security Notes

### XSS
React JSX escaping mitigates stored and reflected XSS by default. No `dangerouslySetInnerHTML` is used. River data displayed on the detail page comes from the scraper (not user input) â€” name strings, numeric levels, dates, status labels. No user-generated content is rendered. Localhost-only, single-user deployment reduces risk to acceptable level for v1.

### CSRF
No mutation endpoints exist in Phase 2 â€” `/api/rivers/:id` is a GET-only endpoint consumed by the detail page. SSE endpoints are read-only. CSRF is not applicable.

### Content Security Policy (CSP)
No CSP header is configured in Phase 2. The detail page renders static data via standard React patterns. Acceptable for v1 localhost usage. For production, consider adding a `default-src 'self'` policy.
