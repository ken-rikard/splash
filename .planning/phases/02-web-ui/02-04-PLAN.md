## Web Security Notes

### XSS
React JSX escaping mitigates stored and reflected XSS by default. No `dangerouslySetInnerHTML` is used. No user-generated content is rendered. The service worker precaches only static build assets (JS, CSS, HTML) — no dynamic content is cached. Localhost-only, single-user deployment further reduces risk.

### CSRF
No mutation endpoints exist in Phase 2. The PWA-related code adds only a service worker for precaching and a manifest for installability — no new API endpoints. CSRF is not applicable.

### Content Security Policy (CSP)
No CSP header is configured in Phase 2. In a PWA context, CSP affects service worker registration scope. For v1 localhost usage this is acceptable. For production, consider adding `default-src 'self'` with `worker-src 'self'` to allow service worker execution.
</threat_model>
