## Web Security Notes

### XSS
React JSX escaping mitigates stored and reflected XSS by default. No `dangerouslySetInnerHTML` is used anywhere in this plan. No user-provided content is rendered тАФ all data is hardcoded sample data. The app runs on localhost only and is single-user, further reducing risk to acceptable level for v1.

### CSRF
This plan produces no mutation endpoints (no POST/PUT/DELETE). All data is hardcoded client-side with no server interaction. CSRF is not applicable.

### Content Security Policy (CSP)
No CSP header is configured in Phase 2. The app runs on localhost only; this is acceptable for v1. For production deployment, consider adding a `default-src 'self'` policy.
