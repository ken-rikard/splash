## 3. Interaction States

### FavoriteButton States

| State | Visual | Behavior |
|-------|--------|----------|

## 5. Accessibility

### FavoriteButton

| Requirement | Implementation |
|-------------|---------------|
| ARIA label | `aria-label={isFavorite ? 'Remove from favorites' : 'Add to favorites'}` — dynamic, reflects current state |
| State announcement | `aria-pressed={isFavorite}` — boolean attribute, screen reader announces "pressed" state |
| Keyboard activation | `Enter` / `Space` — built into shadcn Button (base-ui ButtonPrimitive) |
| Focus indicator | `focus-visible:border-ring focus-visible:ring-1 focus-visible:ring-ring/50` — shadcn ghost default |
| Touch target | `min-h-11` (44px) via shadcn `size-7` (28px) + native button padding? The actual size-7 is 28px, but the button is inside the card layout. Need to ensure: add an explicit `min-h-[44px] min-w-[44px]` wrapper or `p-2` extension for touch accessibility. |
| Color contrast | Unfavorited (`text-slate-400` #94a3b8 on `bg-surface` #111827) → contrast ratio ~5.2:1 ✓ passes AA. Favorited (`text-accent-water` #06b6d4 on `bg-surface` #111827) → ~5.8:1 ✓ passes AA. |
| Prefers-reduced-motion | Animation uses CSS transitions that respond to `prefers-reduced-motion: reduce` — no scale transform when reduced motion enabled |

**Touch target note**: The icon-sm button is 28×28px, which is below the 44×44px minimum for touch. Add a `p-2` (8px padding all around) extension or wrap in a `min-h-[44px] min-w-[44px]` container — tappable area extends beyond the visible button. The existing `.touch-target` class can be applied:

```tsx
<Button variant="ghost" size="icon-sm" className="touch-target">
```

(The `.touch-target` class is already defined in `index.css` as `min-height: 44px; min-width: 44px;`.)

### FilterBar

| Requirement | Implementation |
|-------------|---------------|
| ARIA role | `role="tablist"` on the container, `role="tab"` on each pill |
| ARIA selected | `aria-selected={filter === 'all'}` / `aria-selected={filter === 'favorites'}` |
| ARIA label | `aria-label="Filter rivers by favorites"` on the tablist |
| Keyboard navigation | Arrow keys within tablist for standard tab pattern (Left/Right to switch) — or simplest approach: each pill is a button with `tabindex` management |
| Focus indicator | shadcn ghost focus ring |

### Empty State

| Requirement | Implementation |
|-------------|---------------|
| ARIA live region | `role="status"` or `aria-live="polite"` on the empty state container — announces when filter change shows the empty state |
| Heading hierarchy | Use `<h2>` or keep `<h3>` — the page title is an `<h1>`, empty state should use `<h2>` if it's the primary content |

### RiverCard Favorite Integration

| Requirement | Implementation |
|-------------|---------------|
| Screen reader context | The heart button is inside a `<Link>` (interactive nested in interactive). The `e.preventDefault()` + `e.stopPropagation()` prevents double-activation. Ensure the button has its own accessible name so screen readers don't confuse it with the link. |

## 7. Animation

### Heart Icon Toggle Animation

| Property | Value |
|----------|-------|
| Duration | 150-200ms |
| Easing | `ease-out` (fast start, smooth settle) |
| Transform | `scale(1) → scale(1.2) → scale(1)` |
| Color transition | `150ms ease-out` on `color` + `fill` properties |
| Implementation | CSS `transition` on the Heart SVG element, triggered by class swap |

**CSS approach:**

```tsx
// Tailwind classes on the Heart SVG
<Heart
  className={cn(
    "h-4 w-4 transition-all duration-150 ease-out",
    isFavorite
      ? "text-accent-water fill-current scale-110"
      : "text-slate-400"
  )}
  // The scale is applied via Tailwind's scale-110 when favorited;
  // transition handles the animation between states
/>
```

Wait — `scale-110` is static, not animated. For the "pop" effect, need a keyframe or a brief CSS animation triggered on toggle:

**Better approach — CSS animation triggered on toggle:**

```typescript
// FavoriteButton.tsx
const [animating, setAnimating] = useState(false)

const handleClick = (e: React.MouseEvent) => {
  e.preventDefault()
  e.stopPropagation()
  setAnimating(true)
  onToggle(riverId)
}

// Reset animation state
useEffect(() => {
  if (animating) {
    const timer = setTimeout(() => setAnimating(false), 200)
    return () => clearTimeout(timer)
  }
}, [animating])
```

Tailwind classes:
```
<Heart
  className={cn(
    "h-4 w-4 transition-colors duration-150 ease-out",
    isFavorite ? "text-accent-water fill-current" : "text-slate-400",
    animating && "animate-fav-pop"
  )}
/>
```

New keyframe in index.css:
```css
@keyframes fav-pop {
  0%   { transform: scale(1); }
  40%  { transform: scale(1.25); }
  100% { transform: scale(1); }
}

.animate-fav-pop {
  animation: fav-pop 200ms ease-out;
}
```

And respect reduced motion:
```css
@media (prefers-reduced-motion: reduce) {
  .animate-fav-pop {
    animation: none;
  }
}
```

This is the "subtle CSS transition" approach — the heart pops on toggle with a quick 200ms scale bounce, while the color transition is handled by the existing `transition-colors` class.

### FilterBar Active Tab Transition

```
Active tab change: 150ms ease-out on background-color + text-color
— No slide animation between pills; instant class swap with color transition
```

## 8. Filter Toggle (FilterBar)

### Design Specification

```
│                      └───────────┘  │
```

The FilterBar sits in the same `flex items-center justify-between` row as the "River Levels" page heading. It's a compact inline element, not a standalone bar.

### Layout

```tsx
<div className="flex items-center justify-between mb-8">
  <h1 className="text-3xl font-display font-bold text-white tracking-tight">
    River Levels
  </h1>
  <FilterBar filter={filter} onChange={setFilter} favoritesCount={count} />
```

### FilterBar Component Structure

```tsx
function FilterBar({ filter, onChange, favoritesCount }: FilterBarProps) {
  return (
    <div
      role="tablist"
      aria-label="Filter rivers by favorites"
      className="inline-flex items-center rounded-none border border-white/5 bg-surface p-0.5 gap-0"
    >
      <button
        role="tab"
        aria-selected={filter === 'all'}
        onClick={() => onChange('all')}
        className={cn(
          "inline-flex items-center justify-center px-3 py-1.5 text-[11px] font-semibold uppercase tracking-widest rounded-none transition-colors duration-150 min-h-[44px]",
          filter === 'all'
            ? "bg-surface-elevated text-white"
            : "text-slate-400 hover:text-slate-300"
        )}
      >
        All Rivers
      </button>
      <div className="w-px h-4 bg-white/5" aria-hidden="true" />
      <button
        role="tab"
        aria-selected={filter === 'favorites'}
        onClick={() => onChange('favorites')}
        className={cn(
          "inline-flex items-center justify-center px-3 py-1.5 text-[11px] font-semibold uppercase tracking-widest rounded-none transition-colors duration-150 min-h-[44px]",
          filter === 'favorites'
            ? "bg-surface-elevated text-white"
            : "text-slate-400 hover:text-slate-300"
        )}
      >
        Favorites
        <span className="ml-1.5 text-[10px] text-slate-500">
          {favoritesCount}
        </span>
      </button>
    </div>
  )
}
```

### Badge Count

The favorites count displays next to the "Favorites" label in a smaller muted style (`text-[10px] text-slate-500`). This gives the user a quick sense of how many rivers they're tracking without toggling the filter.

- Format: `Favorites 3` (count inline, non-breaking)
- Hidden when `count === 0` (entire FilterBar hidden)
- The count updates reactively as the user toggles favorites

### Responsive

| Viewport | Behavior |
|----------|----------|
| ≥640px | FilterBar inline with heading, full layout |
| <640px | If the heading + filterbar don't fit in one row, the heading row can stack: heading on top, FilterBar below. Use `flex-wrap` on the container. |

```tsx
// In DashboardPage — the heading container
<div className="flex items-start justify-between mb-8 flex-wrap gap-3">
```

This allows wrapping if viewport is narrow.
