  <action>
    Modify `ui/src/index.css`:

    1. Add keyframe animation at the bottom of the file (after the `.touch-target` class block):

    ```css
    @keyframes fav-pop {
      0%   { transform: scale(1); }
      40%  { transform: scale(1.25); }
      100% { transform: scale(1); }
    }

    .animate-fav-pop {
      animation: fav-pop 200ms ease-out;
    }

    @media (prefers-reduced-motion: reduce) {
      .animate-fav-pop {
        animation: none;
      }
    }
    ```

    2. Do NOT register `animate-fav-pop` in the `@theme` block — it is a utility class, not a theme token. The `@theme` block is for design tokens (colors, fonts, spacing). Animations registered in `@theme` use the `--animate-*` pattern with Tailwind v4, but this phase uses a plain CSS class approach.

    Note: The FavoriteButton component (created in 03-01 Task 1) already references `animating && "animate-fav-pop"` — this task provides the CSS that makes that class work. The animation logic (state management with `animating` flag, `setTimeout` reset, `useEffect` cleanup) is already part of the FavoriteButton component from Plan 03-01. Verify that FavoriteButton has:

    - Click handler sets `setAnimating(true)` before calling `onToggle(riverId)`
    - useEffect resets animating to false after 200ms with setTimeout + clearTimeout cleanup
    - Heart className includes `animating && "animate-fav-pop"`
  </action>
